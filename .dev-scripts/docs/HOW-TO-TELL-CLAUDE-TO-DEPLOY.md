# 🚀 איך להגיד לקלוד לפרוס עבודה?

## 🎯 הנחיות מומלצות לפריסה

### ✅ הנחיה מושלמת - פשוטה וברורה:
```
קלוד, סיימנו עם העבודה.
צור לי commit ו-push ל-main בבקשה.
```

או:

```
קלוד, תעשה commit ו-push את כל השינויים ל-main
```

---

## 📋 מה קלוד יעשה אוטומטית?

כשאתה אומר לי לעשות commit ו-push, אני אעשה את כל השלבים האלה:

### 1️⃣ בדיקות מקדימות
```bash
git status           # בודק מה השתנה
git diff             # מראה את השינויים
git log -5           # מסתכל על commit messages קודמים
```

### 2️⃣ הכנת Commit Message
אני אכתוב הודעה מקצועית שמסכמת:
- מה עשינו (Fix, Feature, Update, Refactor)
- למה עשינו את זה
- קרדיט לקלוד בתחתית

**דוגמה:**
```
✨ FEATURE: הוספת שדה תאריך יעד בהארכת תיק

- הוספת datepicker חדש עם Flatpickr
- עדכון UI בהתאם למסך הקיים
- תיקון בעיית RTL בתאריכים

🤖 Generated with Claude Code
Co-Authored-By: Claude Sonnet 4.5 <noreply@anthropic.com>
```

### 3️⃣ הרצת הפקודות
```bash
git add .
git commit -m "ההודעה שכתבתי"
git push origin main
```

### 4️⃣ אישור
אני אספר לך שהפריסה הצליחה:
```
✅ Commit נוצר בהצלחה
✅ Push ל-main הצליח
✅ Netlify יפרוס את השינויים תוך דקות
```

---

## 🎨 וריאציות של הנחיות

### אם אתה רוצה לפרוס ל-main (Production)
```
קלוד, תעשה commit ו-push ל-main
```
```
סיימנו, תפרוס בבקשה
```
```
הכל מוכן, תעלה את זה
```

### אם אתה רוצה לפרוס ל-feature branch (בטוח יותר)
```
קלוד, תעשה commit ו-push לענף הנוכחי
```
```
תעשה push לענף feature/my-feature
```

### אם אתה רוצה רק commit (בלי push)
```
קלוד, תעשה רק commit בלי push
```
```
צור commit אבל אל תעשה push עדיין
```

---

## ⚠️ אזהרות חשובות

### 🔴 זהיר: Push ל-main = Production מיד!
```
main ← זה פרודקשן! משתמשים יראו את זה תוך דקות
```

אם אתה לא בטוח, **תמיד עדיף לעבוד על feature branch**:
```bash
git checkout -b feature/my-new-feature
```

ואז תגיד לי:
```
קלוד, push לענף feature/my-new-feature
```

## 🧪 תרחישים נפוצים

### תרחיש 1: סיימתי פיצ'ר חדש ורוצה לבדוק לפני פרודקשן

**❌ לא נכון:**
```
קלוד, push ל-main
```

**✅ נכון:**
```
קלוד, אנחנו על feature branch?
אם לא, צור branch חדש feature/extend-deadline-datepicker
ואז push אותו
```

**מה יקרה:**
- אני אבדוק איזה branch אתה עליו
- אם צריך, אצור branch חדש
- אעשה commit ו-push
- Netlify יצור **Deploy Preview** (URL זמני לבדיקה)
- כשתבדוק ותאשר, תגיד לי למזג ל-main

---

### תרחיש 2: תיקנתי באג דחוף - צריך לפרודקשן מיד!

**✅ נכון:**
```
קלוד, זה באג דחוף.
תעשה commit ו-push ישירות ל-main בבקשה.
```

**מה יקרה:**
- אני אעשה commit עם הודעה ברורה (🐛 FIX: ...)
- Push ישירות ל-main
- Netlify יפרוס לפרודקשן תוך 2-3 דקות

---

### תרחיש 3: רוצה לבדוק מה השתנה לפני push

**✅ נכון:**
```
קלוד, תראה לי מה השתנה לפני ה-push
```

**מה יקרה:**
- אני ארוץ `git status` ו-`git diff`
- אראה לך סיכום של כל הקבצים ששונו
- אשאל אותך אם אפשר להמשיך

---

### תרחיש 4: עבדתי על כמה דברים, רוצה לפרוס רק חלק

**✅ נכון:**
```
קלוד, תעשה commit רק של הקבצים הקשורים ל-datepicker
אל תכלול את השינויים ב-clients.html
```

**מה יקרה:**
- אני אעשה `git add` סלקטיבי
- רק הקבצים שציינת יהיו ב-commit
- השאר יישאר uncommitted

## 🎓 טיפים מתקדמים

### טיפ 1: בדוק את הסטטוס לפני
```
קלוד, מה השתנה מהקומיט האחרון?
```

**אני אספר לך:**
- אילו קבצים השתנו
- מה הענף הנוכחי
- האם יש שינויים שלא נשמרו

---

### טיפ 2: צור PR (Pull Request) במקום push ישיר
```
קלוד, צור לי pull request מהענף הזה ל-main
```

**מה יקרה:**
- אני אעשה commit ו-push לענף הנוכחי
- אשתמש ב-`gh pr create` ליצור PR
- תוכל לבדוק את ה-PR לפני שתמזג

---

### טיפ 3: אמור לי מה לכתוב ב-commit message
```
קלוד, עשה commit עם ההודעה:
"הוספת תכונת הארכת תאריך עם datepicker"
```

**אני אשתמש בדיוק בהודעה שלך** (עם הקרדיט שלי בתחתית)

## 📊 טבלת השוואה: מתי לעשות מה?

| מצב | הנחיה לקלוד | מה יקרה |
|-----|-------------|---------|
| 🟢 פיצ'ר חדש (רוצה לבדוק) | "push לענף feature/..." | Deploy Preview בלבד |
| 🔴 באג דחוף | "push ל-main עכשיו" | Production מיד |
| 🟡 לא בטוח | "תראה לי מה השתנה" | סקירה לפני החלטה |
| 🔵 עבודה מתמשכת | "רק commit בלי push" | שמירה מקומית |
| 🟣 מוכן לפרודקשן | "צור PR ל-main" | Pull Request לסקירה |

## 🚨 דברים שאסור להגיד לקלוד

### ❌ לא לעשות:
```
קלוד, תעשה push --force
```
→ **מסוכן!** יכול למחוק היסטוריה

```
קלוד, תעשה git reset --hard
```
→ **מסוכן!** ימחק שינויים לא שמורים

```
קלוד, תדלג על הבדיקות
```
→ **מסוכן!** עלול לשבור את הפרודקשן
