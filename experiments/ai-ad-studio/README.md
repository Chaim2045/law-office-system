## השיטה ב-3 שלבים

### שלב 1 — חומרים (Assets)
לפני שנוגעים בווידאו, בונים ספריית חומרים **עקביים**:

- **מוצר** — תמונה אחת לא מספיקה. הופכים אותה ל"דף מוצר" מלא מכל הזוויות
  (אחרת המודל ממציא מוצר אחר באמצע סצנה).
- **דמות ראשית** — "דף דמות": תקריב פנים + גוף מלא (לגובה ומבנה).
  - בוחרים **2 מועמדות** ולא אחת — מי שמושלמת בסטטי יכולה להתפרק באנימציה.
  - **רקע אפור פשוט** מעלה אחוזי הצלחה.
  - ב-full-body **חותכים את הראש** → פרצוף אחד בלבד למודל להתמקד בו.
- **מיקומים** — קובעים את התחושה. סט פלסטיקי = שום פרומפט לא יציל.
- **אביזרים** — נוצרים ב-AI, לא צריך motion-test.

**עיקרון עלות:** בודקים בתמונות (זול, ~8 תמונות = קרדיט) ולא בווידאו (יקר).
מריצים **2 מיקומים × 2 דמויות = 4 קומבינציות**, אותו פרומפט, משנים משתנה אחד כל פעם.

### שלב 2 — פרומפטים (פה רוב האנשים שורפים קרדיטים)
לא כותבים ביד. בונים **Skill** שמפיק פרומפטים ברמת במאי ל-Seedance 2.
מעלים את החומרים, נותנים לכל אחד **שם + תיאור**. התוצר הוא **מסמך אחד** עם:
**כותרת סגנון** (סגנון / מצלמה / צבע / תאורה) שמודבקת לכל הסצנות —
משנים פעם אחת, משתנה בכל הסרטון.

→ זה בדיוק מה ש-`seedance-director-skill.md` עושה.

### שלב 3 — ייצור הסצנות
- הניסיון הראשון כמעט אף פעם לא עובד. מתקנים **שורה אחת** ומג'נרטים מחדש.
- **אסור פעולות כלליות** — מפרקים תנועה-תנועה (beats), חושבים כמו במאי.
- **סטוריבורד לפני וידאו** — סקיצה ב-GPT Image מתארת סצנה טוב יותר מטקסט.
- דמות שמשתנה תוך כדי (רץ → מזיע) → דף-דמות נוסף, לא נלחמים ברפרנס יחיד.

## איך משתמשים בפרוטוטייפ

1. ממלאים brief לפי התבנית ב-`brief-template.md`.
2. מדביקים את `seedance-director-skill.md` כ-system prompt ב-Claude (claude.ai או Claude Code),
   מצרפים את ה-brief ותיאורי החומרים.
3. מקבלים מסמך-בימוי: כותרת סגנון + רישום חומרים + shot list עם beats + תיאורי סטוריבורד.
4. מייצרים סקיצות (GPT Image) → ואז וידאו (Seedance) shot-by-shot.
5. מתקנים שורה-שורה. שומרים את ה-takes הכי טובים.

ראו `example-run.md` להרצה מלאה לדוגמה (פרסומת משקה ספורט, 15 שניות).
