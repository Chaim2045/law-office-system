# Rubric — PR-G.3.13: Fix Report Modal hours display + status badge

**Scope:** Admin Panel `ClientReportModal` shows misleading data: archived services rendered as open (no status badge), overdraft services rendered as having hours remaining (clamped to 0 + name-matching recalc fails when `service.name` is undefined). Single-file fix in `apps/admin-panel/js/ui/ClientReportModal.js`. Tactical-first approach approved by Haim after `devils-advocate` + `frontend-ui-expert` both recommended against bigger refactor (4,208-line unified-modal proposal).

**Triggering case:** Client `2025724` (PROD) — service `service_1773822736935` (`status='active'`, `totalHours=34`, `hoursUsed=42.93`, `overrideActive=true`, `hoursRemaining=-8.93`) is displayed by Report Modal as if hours remain. Service `srv_1766392900070` (`status='archived'`) is displayed as open. Management Modal renders both correctly via `getServiceStatusBadge` and direct reads of `service.hoursUsed`.

**SSOT citation:** `SYSTEM_STATUS.md:251-254` — `service.hoursUsed` is the canonical value maintained by `functions/shared/aggregates.js:43-73`. UI MUST read this field, not recompute.

**App + Env:** Admin Panel | DEV first → PROD via `production-stable` merge after Haim approval.
