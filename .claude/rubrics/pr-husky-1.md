# Rubric — PR-HUSKY-1: husky v9→v10 forward-compat (`prepare` + hook boilerplate)

**Title:** chore(tooling): migrate `prepare` to bare `husky` and strip deprecated v9 hook boilerplate (forward-compat for husky v10).
**Branch:** `chore/husky-v10-forward-compat`
**Base:** `main`
**App / Env:** dev tooling only (git hooks + npm `prepare`). No app code, no Cloud Functions, no Firestore, no deploy, nothing customer-facing. Local + CI (`npm ci`) only.
**Effort:** LIGHT — one-line `package.json` change + removal of 2 boilerplate lines from each of 2 hook files. Mechanical, per husky's own v9→v10 migration guidance. effort-scaler skipped (obviously LIGHT); outcomes-grader skipped (<50 lines, no logic). Haim approved "רוּבריקה קלה" at checkpoint.

**Why:** husky 9.1.7 prints `husky - install command is DEPRECATED` on every `npm install`/`npm ci` (the `prepare` script runs `husky install`), and the generated hooks still source `_/husky.sh` via the deprecated boilerplate husky warns "WILL FAIL in v10.0.0". When Dependabot bumps husky to 10.x, `husky install` is removed and the boilerplate hard-fails — breaking `npm ci` locally and in every CI job that runs it. This PR makes that bump a no-op instead of a breakage.

**Scope (3 mechanical changes):**
1. `package.json` `prepare`: `"husky install"` → `"husky"` (bare `husky` is the modern init; works on v9 and v10).
2. `.husky/pre-commit`: remove the 2 deprecated lines (`#!/bin/sh` + `. "$(dirname "$0")/_/husky.sh"`). Body (`npx lint-staged`) unchanged.
3. `.husky/pre-push`: same 2-line removal. The 100+ line branch-aware prod-safety body unchanged.

**Explicitly OUT of scope (documented, not dropped):**
- husky is NOT bumped to 10.x here. The fix is forward-compatible (works on both 9.1.7 and 10.x); the actual bump arrives via Dependabot and this PR ensures it lands clean. Bumping now would be an independent dependency change with its own review surface.
- `.husky/_/` (husky-managed, gitignored `*`) is untouched — regenerated by the `prepare` script; never tracked.
- The uncommitted `ci-cd-production.yml` change in the working tree (**PR-CI-2** — the deferred script-injection follow-up flagged out-of-scope in `pr-ci-1.md`) is a SEPARATE task, left untouched and unstaged. Flagged to Haim.
