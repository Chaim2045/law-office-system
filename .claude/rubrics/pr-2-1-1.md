# Rubric — PR-2.1.1

**Title:** feat(shared): canonical service-classification module (PR-2.1.1)
**Branch:** `feat/shared-business-rules-pr-2-1-1`
**Base:** `main`
**Scope:** Create a canonical service-classification module (`shared/business-rules/service-classification.js`) and distribute it to all three codebases (Functions, Admin Panel, User App) using the **PR-G.2 mirror+sync-test pattern**. Make `functions/shared/aggregates.js` re-export from its own in-tree mirror at `functions/shared/business-rules/service-classification.js`. Preserve the existing `isFixedService` definition in `apps/admin-panel/js/core/client-type-display.js:29` (existing Vitest unit tests depend on it) — the sync test enforces it stays in lockstep with the canonical. The behavior of `isFixedService` is bit-for-bit identical to today; this PR only consolidates the definition into a single SOT with a CI gate against drift.

## Risk profile

**Low–Medium.** Additive new module + replacement of one duplicate + ESLint config widening. No business logic changes. Three real concerns:
1. Module load order in browser — adapter must load before `client-type-display.js`.
2. Sync-test wiring — must actually run in Vitest CI (the legacy PR-G.2 sync test is silently bypassed; this PR creates a properly-wired one).
3. ESLint config widening — removing `functions/**` + `scripts/**` from ignores may surface pre-existing lint errors that block the PR. Pre-PR action: dry-run lint, document any pre-existing errors that require a separate cleanup PR.

## MUST criteria (block on FAIL)

### M1 — Canonical module created with correct API surface
**Rule:** `shared/business-rules/service-classification.js` exists. Exports exactly three predicates plus the constants used internally:
- `isFixedService(svc)` — returns `true` iff `svc.type === 'fixed'` OR (`svc.type === 'legal_procedure'` AND `svc.pricingType === 'fixed'`)
- `isHourlyService(svc)` — returns `true` iff `svc.type === 'hours'` OR (`svc.type === 'legal_procedure'` AND `svc.pricingType === 'hourly'`)
- `isLegalProcedureService(svc)` — returns `true` iff `svc.type === 'legal_procedure'`
- All three return `false` on `null`, `undefined`, or service objects missing the `type` field. No throws on bad input.
**Evidence required:** Module file + unit test asserting all 4 service shapes + null/undefined behavior.
