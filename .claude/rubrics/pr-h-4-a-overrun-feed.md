## MUST criteria (block on FAIL)

### M1 — 🔴 Canonical budget-meter helper + Haim-locked thresholds
**Rule:** A NEW pure helper `apps/admin-panel/js/core/budget-status.js` (NO DOM, NO Firebase; dual-export `window.BudgetStatus` + CommonJS), mirroring `profitability-format.js`'s `hoursStatus`. `budgetStatus(actualMinutes, estimatedMinutes) → { level, label, percent, isOver }`: no positive-finite estimate → `{neutral, 'אין תקציב', null, false}`; `percent = round(actual/estimate*100)` (actual→0 if not finite); `< 85` → success 'בתקציב'; `85..100` → warning 'מתקרב לתקציב'; `> 100` → danger 'חריגת תקציב' (isOver:true). The 85/100 thresholds are the single source the admin feed (and PR-b's worker surface) align to. Also exports an hours formatter (minutes/60, 1 decimal).
**Evidence:** `budget-status.js`; `tests/unit/admin-panel/budget-status.test.ts` (no/zero/negative estimate → neutral; <85 success; exactly 85 + 100 warning; >100 danger+isOver; percent rounding; the hours formatter) — root vitest green.
