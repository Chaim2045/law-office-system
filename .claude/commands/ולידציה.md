# ולידציה — ALIAS ל-/validate-strict

פקודה זו היא alias בעברית לנוחות.
ההתנהגות זהה ל-`/validate-strict`.

## מה קורה כשאתה מפעיל /ולידציה:

1. **הפעלה:** הסוכן `prod-gatekeeper` מופעל ב-Zero Context
2. **בדיקות חובה:**
   - ✅ DEV smoke test עבר (URL נבדק בפועל)
   - ✅ Cache-bust בוצע לפני בדיקת PROD
   - ✅ `npm run type-check` עובר
   - ✅ `npm run lint` עובר
   - ✅ `npm run test` עובר
   - ✅ `git status` נקי — אין שינויים לא-commited
   - ✅ Current branch תואם את ה-environment (main=DEV, production-stable=PROD)
   - ✅ Functions logs — אפס errors בדקה האחרונה
   - ✅ Console ב-DEV — נקי לחלוטין
3. **תוצאה:** VERDICT — `PASS` או `FAIL`
   - אם FAIL — רשימת הבדיקות שנכשלו + מה לתקן
   - אם PASS — אישור ל-merge ל-production-stable

## שימוש:
```
/ולידציה                   # סביבה נוכחית
/ולידציה prod              # pre-flight ל-PROD
/ולידציה dev               # smoke ל-DEV בלבד
```

## חוקי ברזל (מ-CLAUDE.md):
- כל שינוי חייב לעבור DEV לפני PROD
- Cache-bust חובה לפני בדיקת PROD
- Smoke test חובה אחרי deploy
- כל console error = FAIL
- אסור לעקוף branch protection

## מעבר להוראות המלאות:
ראה `.claude/commands/validate-strict.md` לפרוטוקול המלא.
