# ביקורת — ALIAS ל-/review-strict

פקודה זו היא alias בעברית לנוחות.
ההתנהגות זהה ל-`/review-strict`.

## מה קורה כשאתה מפעיל /ביקורת:

1. **הפעלה:** הסוכן `code-reviewer` מופעל במצב בידוד
2. **קלט:** diff הנוכחי / PR מספר שציינת / קובץ שצוין
3. **שלבים:**
   - Stage 1 — Formatting (CRLF/LF, indentation, line endings)
   - Stage 2 — Linting (ESLint, Stylelint, TypeScript)
   - Stage 3 — Security (XSS, SQL injection, hardcoded secrets, auth gaps)
   - Stage 4 — Data Integrity (Transactions, Idempotency, SSOT violations)
   - Stage 5 — Performance (bundle size, query efficiency, cold starts)
   - Stage 6 — Architecture (SOLID, DRY, SSOT enforcement)
4. **תוצאה:** VERDICT — `PASS` או `FAIL`
   - אם FAIL — כל ממצא עם מספר שורה + הצעה לתיקון
   - אם PASS — אישור להמשיך ל-`/ולידציה` (אם זה לפני PROD)

## שימוש:
```
/ביקורת                    # על ה-diff הנוכחי
/ביקורת 142               # על PR #142
/ביקורת src/module.ts      # על קובץ ספציפי
```

## מעבר להוראות המלאות:
ראה `.claude/commands/review-strict.md` לפרוטוקול המלא.
