# ניווט — תהליך עבודה

פקודה זו מפעילה את הסוכן `navigator-process-guide`.

## מתי להשתמש:
- שכחת באיזה שלב אתה בפרוטוקול
- לא בטוח איזה סוכן מתאים עכשיו
- התחלת משימה חדשה ואתה לא יודע מאיפה להתחיל
- נתקעת בשלב מסוים ולא יודע איך להתקדם

## פורמט הפלט של Navigator:
```
📍 איפה אתה עכשיו:
   Stage: [Intent / Investigation / Checkpoint / Planning / Code / Gates]
   App: [User App / Admin Panel / Functions / Shared]
   Env: [DEV / PROD]
   Branch: [current-branch-name]

✅ מה כבר נעשה:
   - [רשימת שלבים שהושלמו]

➡️ הצעד הבא המומלץ:
   /[command] [args]
   או
   Agent: [agent-name]
   הסבר: [למה זה הצעד הנכון]

⚠️ Checkpoints שעדיין חסרים:
   - [אם יש]
```

## מפת הפרוטוקול הכללית (מ-CLAUDE.md):
1. **Intent** — חיים מגדיר מה רוצים (עם /טומי או intent-refiner)
2. **Investigation** — מיפוי הבעיה (עם /ארכיטקט) — READ ONLY
3. **Checkpoint** — חיים מאשר או שולל
4. **Planning** — תכנון עם אופציות (עם /תכנון)
5. **Code** — הקוד (עם backend/frontend/etc)
6. **Gates** — ביקורת + ולידציה (עם /ביקורת → /ולידציה)

**חובה לעבור בסדר. אסור לדלג על Checkpoint.**
