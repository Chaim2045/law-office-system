# NEW BRANCH — SAFE BRANCH CREATION

## ROLE
אתה יוצר ענף עבודה חדש לפי הכללים של הפרויקט.

## REQUIRED FIRST STEP
לפני יצירת ענף, אשר:
1. שם הענף (באנגלית, עם מקפים)
2. מה המטרה (תיקון / פיצ'ר / חקירה / ניקוי)
3. מאיזה ענף יוצאים (תמיד main אלא אם נאמר אחרת)

## NAMING CONVENTION
- תיקון באג: `fix/תיאור-קצר`
- פיצ'ר חדש: `feat/תיאור-קצר`
- חקירת נתונים: `investigate/תיאור-קצר`
- ניקוי / שיפור: `chore/תיאור-קצר`
- ביצועים: `perf/תיאור-קצר`
- בדיקות: `test/תיאור-קצר`

## EXECUTION
1. `git checkout main`
2. `git pull origin main`
3. `git checkout -b <branch-name>`
4. אשר למשתמש שהענף נוצר

## HARD RULES
- ענף חדש תמיד יוצא מ-main (= DEV)
- אסור לצאת מ-production-stable
- אסור ליצור ענף בלי שם ומטרה ברורים
- אסור לעשות push בלי אישור מפורש

## STYLE
- עברית בלבד
- ישיר
- קצר
