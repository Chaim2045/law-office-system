# סנכרון - משיכה מ-GitHub

⚠️ **אזהרה:** תהליך זה ימחק שינויים מקומיים שלא נשמרו!

בצע את הפעולות הבאות **בדיוק לפי הסדר**, ועצור מיד אם יש שגיאה:

1. הרץ: `git status` - בדוק אם יש שינויים לא שמורים
   - **אם יש שגיאה - עצור והצג את השגיאה למשתמש. אל תמשיך.**

2. **הצג למשתמש את הפלט של git status**

3. **אם יש שינויים לא שמורים (modified/untracked files):**
   - **עצור והצג אזהרה למשתמש:** "⚠️ יש שינויים לא שמורים במחשב הזה! אם תמשיך, השינויים האלה יימחקו. האם להמשיך? (כן/לא)"
   - **אל תמשיך ללא אישור מפורש מהמשתמש**

4. **רק לאחר אישור מפורש מהמשתמש** - הרץ: `git fetch origin`
   - **אם יש שגיאה - עצור והצג את השגיאה למשתמש. אל תמשיך.**

5. הרץ: `git reset --hard origin/main` - סנכרן עם GitHub (מוחק שינויים מקומיים)
   - **אם יש שגיאה - עצור והצג את השגיאה למשתמש. אל תמשיך.**

6. הרץ: `git clean -fd` - נקה קבצים שלא במעקב
   - **אם יש שגיאה - עצור והצג את השגיאה למשתמש.**

✅ **רק אם הכל עבר בהצלחה** - אשר למשתמש: "הפרויקט עודכן בהצלחה מ-GitHub. כל הקבצים מסונכרנים."

⛔ **חשוב:**
- אל תנסה לפתור שגיאות לבד
- אל תריץ פקודות מסוכנות (reset/clean) ללא אישור מפורש
- אם משהו לא עובד - דווח למשתמש ועצור
- אל תמחק נתונים ללא אישור
