# .claude/audits/

תיקייה ייעודית לחבילות ביקורת על הסוכנים, הפקודות, הפרוטוקולים וה-CLAUDE.md.

## מטרה
בכל פעם שמריצים ביקורת חיצונית (פרקליט שטן, code review של הארכיטקטורה, התייעצות עם מפתח חיצוני וכד') — התוצאה נשמרת כאן.

## פורמט שמות
```
YYYY-MM-DD-<תיאור>-<סוג>.md
```

דוגמאות:
- `2026-04-19-AUDIT-PACKAGE.md` — חבילת מקור לביקורת
- `2026-04-19-OPENING-PROMPT.md` — prompt להדבקה בסשן חיצוני
- `2026-04-25-devils-advocate-findings.md` — ממצאים שחזרו מהביקורת

## כללים
- **אין לערוך חבילה שכבר נסגרה** — היא מתעדת מצב בזמן מסוים. יצירת גרסה חדשה = קובץ חדש עם תאריך חדש.
- **אין לתקן סוכנים על סמך חבילה פתוחה.** קודם מסיימים את הביקורת, ורק אחר כך מתקנים (ב-branch נפרד).
- **תיעוד של מה שנעשה אחרי כל ביקורת** — בקובץ נפרד `YYYY-MM-DD-actions-taken.md`.
