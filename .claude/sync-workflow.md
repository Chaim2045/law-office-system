# תהליך סנכרון בין מחשבים - הנחיות לקלוד

## 📤 כשהמשתמש אומר: "עדכן לעבודה" / "push לעבודה" / "סיימתי בבית"

**מחשב נייד (בבית) → GitHub**

הרץ את הפקודות הבאות בסדר:

```bash
# 1. בדוק מצב נוכחי
git status

# 2. הוסף את כל השינויים
git add .

# 3. צור commit עם תיאור אוטומטי
git commit -m "עדכון מהבית - $(date +%Y-%m-%d_%H-%M)"

# 4. דחוף ל-GitHub
git push origin main
```

✅ **אשר למשתמש:** "הפרויקט עודכן בהצלחה ל-GitHub. אתה יכול למשוך אותו במחשב בעבודה."

---

## 📥 כשהמשתמש אומר: "משוך מהבית" / "pull מהבית" / "הגעתי לעבודה"

**GitHub → מחשב בעבודה**

⚠️ **אזהרה ראשונה:** בדוק אם יש שינויים לא שמורים:

```bash
git status
```

אם יש שינויים שלא בוצע עליהם commit:
- **שאל את המשתמש:** "יש שינויים לא שמורים במחשב בעבודה. האם לשמור אותם לפני המשיכה? (כן/לא)"
  - אם כן: הרץ `git add . && git commit -m "שמירה אוטומטית לפני משיכה"`
  - אם לא: המשך

```bash
# 1. הורד את העדכונים האחרונים
git fetch origin

# 2. סנכרן עם הגרסה מ-GitHub (מוחק שינויים מקומיים)
git reset --hard origin/main

# 3. נקה קבצים שלא במעקב
git clean -fd
```

✅ **אשר למשתמש:** "הפרויקט עודכן בהצלחה מ-GitHub. כל הקבצים מסונכרנים."

---

## 🔄 כשהמשתמש אומר: "הפוך" / "עדכן לבית" / "סיימתי בעבודה"

**מחשב בעבודה → GitHub → מחשב נייד**

אותו תהליך כמו "עדכן לעבודה" (push), ואז במחשב נייד אותו תהליך כמו "משוך מהבית" (pull).

## ⚠️ כללים חשובים:

1. **תמיד הצג את פלט git status לפני פעולות**
2. **תמיד אשר למשתמש אחרי השלמת התהליך**
3. **אם יש שגיאות - הסבר במפורש מה הבעיה וכיצד לפתור**
4. **אל תריץ git reset --hard ללא בדיקה ואזהרה מראש**
5. **אם יש קונפליקטים - עצור והסבר למשתמש**

---

## 🚨 טיפול בשגיאות נפוצות:

### שגיאה: "Your branch is behind"
```bash
git pull origin main --rebase
```

### שגיאה: "Merge conflict"
```bash
# הצג את הקבצים הבעייתיים
git status

# הסבר למשתמש איזה קבצים מתנגשים וכיצד לפתור
```

### שגיאה: "Permission denied"
```bash
# בדוק אם המשתמש מחובר ל-Git
git config user.name
git config user.email
```
